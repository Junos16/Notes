# Motivation
- A memory unit in a computer helps it store data, instructions, addresses for the program that is being executed to access
- In an ideal world a computer would have access to infinite amounts of storage so that it can handle any program no matter the amount of memory it requires
- The main issue with having a large (if not infinite) memory unit is that it becomes more challenging to retrieve data
	- In case of a small memory unit, searching for any one item is quicker due to the limited number of locations to be searched

# Cache
## Principle of Locality
- Only a small amount of data stored in the memory is relevant to a program being executed by the computer at a time
- ***Temporal Locality:*** If a program accesses an item in memory recently, it would most likely reference it again soon
	- Example: During the execution of for loops the same instructions and the same data is being accessed over and over
- ***Spatial Locality:*** If a program access an item in memory, then items stored by close that most likely are related to that item would be referenced by the program
	- Example: While performing operations
