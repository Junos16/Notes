>81. Given a set of points where most points are far apart, but a few points are very close to each other, can you develop an algorithm more efficient from our original algorithm to find the closest pair in this special case.

We can create a grid for the whole space of points with cells of size dxd where d is chosen using some heuristic. Points that we want to compare now lie in these cells and the denser points lie in the same or neighbouring cells. Whereas sparser points lie in cells without any other points in it or with neighbouring cells with points.

We now maintain a global minimum distance which is initially a large value and update it gradually. For a point in a cell we find the distance between it and other points in the same cell or the 8 neighbouring cells around it. This gives us the shortest pair after all of the cells have been traversed. 

For each point that we check, we are making a limited number of comparisons with points in the same and neighbouring cell. For sparse points we have make no comparisons. If we have a good heuristic for choosing a value for d we can make all the comparison for the denser points in constant time. 

Setting up the grid can be done in O(n) and so can all the comparisons for each point. So the overall algorithm is in O(n)
